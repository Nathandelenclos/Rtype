1 # Docker Image Availability Post Project Linking
2 
3 ## Understanding the Workflow of `link-project` Job
4 
5 ### Overview
6 The `link-project` job in Jenkins not only sets up projects for continuous integration but also plays a crucial role in the continuous deployment process, particularly in the context of Docker images.
7 
8 ### Post-Linking Actions
9 - **Job**: `link-project`
10 - **Functionality**: Upon successfully linking a project, this job triggers a series of actions, including the building and pushing of the project's Docker image to a specified Docker registry.
11 
12 ### Docker Image Availability
13 - **Registry URL**: Specified in the `link-project` job parameters (e.g., `REGISTRY_URL: "{{registry_domain}}"`).
14 - **Image Accessibility**: Once the `link-project` job completes its execution, the Docker image of the linked project is built and pushed to the specified Docker registry.
15 - **Use Case**: This allows the Docker image to be readily available for deployment, testing, or further development stages.
16 
17 ## Accessing the Docker Image
18 1. **Locate the Image**: The Docker image can be found in the Docker registry specified in the `REGISTRY_URL` parameter.
19 2. **Pulling the Image**: Users can pull the Docker image from the registry using Docker commands, for example, `docker pull registry.example.com/project-image`.
20 3. **Deployment and Usage**: The pulled image can be used for deployment in various environments, further ensuring the consistency and reliability of the application.
