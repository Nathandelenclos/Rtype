15 ## Introduction
16 
17 For this Advanced C ++ Knowledge Unit project, R-Type will present the development of network video games.
18 Implementation of a multi-threaded server and a graphical client, using a reusable game engine of our own design.
19 The game should also be fun to play!
20 The purpose of this project is to create a one-to-four player game, using a client/server architecture.
21 This is important. It MUST be a client/server architecture. Peer-to-peer communication is not allowed.
22 
23 ## Goal of the project
24 
25 The goal of this project is to create a multi-player video game based on the rules of the R-Type game.
26 The game must be playable in local and network mode.
27 The game must be composed of a server and a client.
28 The server must be able to manage several simultaneous game rooms.
29 The client must be able to display the game in real time.
30 The client and the server must be able to communicate using the UDP protocol.
31 The server and the client must be able to run on multiple operating systems (Windows, Linux, MacOS).

98 ## ECS Engine
99 
100 Entity Component System (ECS) is a software architectural pattern mostly used in video game development for the representation of game world objects.
101 An ECS comprises entities composed from components of data, with systems which operate on entities' components.
102 
103 ECS follows the principle of composition over inheritance, meaning that every entity is defined not by a type hierarchy, but by the components that are associated with it.
104 Systems act globally over all entities which have the required components.

106 ## Network UDP Protocol
107 
108 The User Datagram Protocol (UDP) is one of the core members of the Internet protocol suite.
109 The protocol was designed by David P. Reed in 1980 and formally defined in RFC 768.
110 
111 The role of this protocol is to allow the transmission of data (as datagrams) in a very simple way between two entities, each being defined by an IP address and a port number.
112 No prior communication is required to establish the connection
113 
114 Then the UDP protocol is used to communicate between the server and the client,
115 the server sends the client the information it needs to display the game and the client sends the server the actions of the player.
